So is a container a virtual environment?

Kind of but not quite. A Virtual Environment is primarily used for creating an isolated environment to develop within wherease a container is more about bundling everything an app needs into a self contained package

Virtual environments virtualise the entire stack down to the kernel. Containers are more lightweight and virtualise less of the stack (down to the OS only). Containers are less about fencing off and more about creating a layer of abstraction around each different application to simplfy and standardise the host machine's duties
